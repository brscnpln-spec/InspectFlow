Context:
Mevcut uygulamaya yalnızca admin kullanıcıların görebileceği bir Notification Center ekliyoruz. Bildirimler sağ üstte, kullanıcı emailinin yanında bir bell icon ile gösterilecek. Her bildirim tıklanabilir olacak ve ilgili deep link’e yönlendirecek.

Task:
Admin-only Notification Center implement et.

Requirements:

1. Visibility
- Notification Center sadece admin kullanıcılar için görünecek.
- Service member veya diğer roller bell icon’u ve bildirimleri görmemeli.
- Bu kontrol hem UI hem de backend/data access seviyesinde uygulanmalı.

2. UI Placement
- Sağ üst header alanında, kullanıcı emailinin yanına bell icon ekle.
- Okunmamış bildirim varsa bell icon üzerinde badge/count göster.
- Bell icon’a tıklanınca dropdown / popover şeklinde Notification Center açılsın.
- Notification listesi modern, sade ve mevcut design system ile uyumlu olsun.

3. Notification Types & Deep Links
Aşağıdaki bildirim tiplerini destekle:

- Team Member inspection’ı kabul etti
  - Link: inspection detail

- Team Member inspection’ı belirlenen süre içinde kabul etmedi
  - Link: inspection detail

- Team Member inspection’ı tamamladı
  - Link: inspection detail

- Inspection günü geçti fakat max 3 iş günü içinde inspection sistemde tamamlanmadı
  - Link: inspection detail

- Bir feedback formu dolduruldu
  - Link: Feedback Manager

- Bir inspection’a ait feedback formunu belirlenen süre dahilinde hiç kimse doldurmadı
  - Link: inspection detail

- Başka bir kullanıcı yeni bir Tenant ekledi
  - Link: Tenants

- Ertesi gün bir inspection var
  - Link: Calendar

4. Notification Data Model
Gerekirse notifications tablosu oluştur:
- id
- type
- title
- message
- target_url
- related_inspection_id nullable
- related_tenant_id nullable
- related_feedback_id nullable
- created_at
- read_at nullable
- is_read boolean
- recipient_role / admin-only marker

5. Behavior
- Bildirime tıklanınca:
  - ilgili target_url’e navigate et
  - bildirimi read olarak işaretle
- Okunmamış bildirim sayısı badge’de gösterilsin.
- Bildirimler newest first sıralansın.
- Read / unread görsel olarak ayrışsın.
- “Mark all as read” aksiyonu ekle.
- Empty state göster: “No notifications yet.”

6. Notification Generation Logic
Aşağıdaki event’lerde notification oluştur:

- Team member inspection’ı kabul ettiğinde
- Team member belirlenen süre içinde inspection’ı kabul etmediğinde
- Team member inspection’ı tamamladığında
- Inspection date geçmiş ve max 3 iş günü içinde sistemde tamamlanmamışsa
- Feedback formu doldurulduğunda
- Feedback formu belirlenen süre içinde hiç kimse tarafından doldurulmamışsa
- Admin dışında başka bir kullanıcı yeni Tenant eklediğinde
- Inspection tarihinden 1 gün önce reminder notification oluşturulduğunda

7. Scheduling / Time-Based Notifications
- Süre bazlı kontroller için mevcut scheduler/cron/edge function yapısını incele.
- Eğer yoksa Supabase scheduled function / cron-compatible approach ile tasarla.
- Aynı notification’ın tekrar tekrar oluşmasını engellemek için idempotency kontrolü ekle.
- Örn: aynı inspection + notification type için duplicate notification oluşturma.

8. Permissions / Security
- Admin kullanıcılar tüm notification’ları görebilir.
- Admin olmayan kullanıcılar notifications tablosuna erişememeli.
- RLS policies veya backend authorization logic ekle/güncelle.
- Notification creation logic güvenli şekilde server-side çalışmalı.

9. Constraints
- Mevcut auth ve role management yapısını bozma.
- Header layout’unu gereksiz yere değiştirme.
- Service member UI’ına notification center ekleme.
- Existing inspection, tenant, feedback ve calendar flow’larını bozma.
- “Notification Center” sadece admin için çalışmalı.

10. Implementation Approach
Önce mevcut header, auth role yapısı, inspection status flow, feedback flow, tenant create flow ve calendar data yapısını analiz et.
Ardından minimum gerekli değişikliklerle:
1. Notification data model
2. Admin-only RLS / permission logic
3. Notification generation event hooks
4. Time-based scheduler logic
5. Header bell icon + dropdown UI
6. Deep link navigation + read/unread behavior
implement et.

Goal:
Admin kullanıcıların önemli inspection, feedback, tenant ve calendar olaylarını tek bir notification center üzerinden takip edebilmesini sağlamak.
